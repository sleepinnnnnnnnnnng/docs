## 社区角色

NEC 社区采用分层贡献者模型，角色随着参与深度逐步晋升：

| 角色 | 定义 | 典型职责 |
| :--- | :--- | :--- |
| **访客（Visitor）** | 初次接触社区，尚未提交贡献 | 浏览文档、参与公开讨论 |
| **成员（Member）** | 完成新人 onboarding，至少提交过一次有效 PR/Issue | 参与小组任务、更新个人进度 |
| **贡献者（Contributor）** | 持续参与项目或竞赛模块 | 提交代码/文档、Review PR、协助新人 |
| **维护者（Maintainer）** | 对某个 SIG 或模块有长期投入与决策权 | 维护代码库、评审重要变更、组织周会 |
| **核心治理组（Core）** | 社区创始维护者与各 SIG 负责人 | 制定路线图、处理冲突、代表社区对外合作 |

## 决策流程

重要事项遵循「提议 → 讨论 → 决议 → 执行 → 复盘」的闭环：

1. **提议（Propose）**：任何成员可通过 Issue、飞书文档或周会提出改进建议。
2. **讨论（Discuss）**：相关 SIG 在 3–7 天内公开讨论，收集反对意见与替代方案。
3. **决议（Resolve）**：
   - 日常事务：对应 SIG Maintainer 拍板；
   - 跨 SIG 或重大变更：核心治理组 consensus，必要时投票。
4. **执行（Execute）**：明确负责人与截止日期，更新到 Roadmap 或任务看板。
5. **复盘（Review）**：完成后在例会或文档中记录结果与经验教训。

## 会议与沟通节奏

- **周会**：每周固定时间线上同步，各 SIG 汇报进度、风险与下周计划。
- **季度复盘**：回顾 Roadmap 完成情况，调整优先级与资源配置。
- **公开记录**：会议纪要整理后归档到飞书 Wiki，并在社区文档中保留入口。
