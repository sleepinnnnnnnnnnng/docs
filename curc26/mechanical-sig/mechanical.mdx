## 三、文件结构规范 (ROB_CTRL)

### 1. 项目顶层命名
推荐使用 **`ROB_CTRL`** (Robot + Control)，直观体现机电一体化。

### 2. 一级结构 (项目主干)
```text
/ROB_CTRL
 ├─ 00_ADMIN   （项目管理/计划）
 ├─ 01_REQ     （需求&接口）
 ├─ 02_ARCH    （系统架构）
 ├─ 03_MECH    （机械设计 ★核心）
 ├─ 04_ELEC    （电气/控制）
 ├─ 05_SW      （软件/PLC）
 ├─ 06_SIM     （运动/强度仿真）
 ├─ 07_MFG     （制造&工艺）
 ├─ 08_ASSY    （整机装配）
 ├─ 09_TST     （调试/验收）
 └─ 99_REF     （标准/外部资料）
```

### 3. 机械设计 (03_MECH)
深度适配 SolidWorks (参数化) + Rhino (曲面/外形) 工作流。

```text
/03_MECH
 ├─ 01_CON     （方案/机构草案）
 ├─ 02_LAYOUT  （总体布局）
 ├─ 03_CAD     （建模核心目录）
 ├─ 04_CAL     （计算选型）
 ├─ 05_DWG     （工程图）
 ├─ 06_BOM     （机械BOM）
 └─ 07_REV     （变更记录）
```

### 4. CAD 文件夹详解
**SolidWorks (参数化主力)**
```text
/03_CAD/SW
 ├─ PRT       （零件 .SLDPRT）
 ├─ ASM       （装配 .SLDASM）
 ├─ DRW       （工程图 .SLDDRW）
 ├─ TPL       （模板）
 └─ LIB       （标准件/Toolbox）
```

**Rhino (外形 / 曲面 / 防护罩)**
```text
/03_CAD/RH
 ├─ SURF      （曲面）
 ├─ SHAPE     （外形方案）
 ├─ EXPORT    （STEP/IGES 导出给SW使用）
```
> **规则**：Rhino 只做外形 & 曲面，最终结构 **必须回到 SW 主装配**，使用 `EXPORT` 明确跨软件边界。

### 5. 机器人专用机构拆分 (建议)
```text
/03_CAD/SW/ASM
 ├─ BASE       （底座）
 ├─ AX1        （J1 旋转轴）
 ├─ AX2        （J2 俯仰）
 ...
 ├─ EOAT       （末端执行器）
 └─ CABLE      （拖链/线缆）
```

## 四、机电协作与制造

### 1. 工业控制 & 机电接口
- **/01_REQ/IO_MAP**: IO点位分配表
- **/01_REQ/IF_ME**: 机电接口定义（安装孔位、走线空间）
- **/04_ELEC**: 电气原理图与电柜布局

### 2. 制造 & 装配 (落地必备)
- **/07_MFG**: 分类存放 CNC、钣金 (SHT)、3D打印 (RP) 等加工文件。
- **/08_ASSY**: 存放装配 SOP、工装设计、螺丝力矩表。

## 五、命名与缩写规范

### 1. 统一缩写速查
| 缩写 | 含义 | 缩写 | 含义 |
| :--- | :--- | :--- | :--- |
| **ROB** | 机器人 | **IF** | 接口 |
| **AX** | 轴 (Axis) | **IO** | 输入输出 |
| **EOAT** | 末端执行器 | **MFG** | 制造 |
| **CAL** | 计算 | **ASSY** | 装配 |
| **SIM** | 仿真 | **REV** | 版本 |

### 2. 命名示例
- 零件：`AX2_BRACKET_LH_V1.3.SLDPRT` (部位_名称_方位_版本)
- 装配：`ROBOT_MAIN_ASM.SLDASM` (主装配)

## 六、工作原则 (Best Practices)
1. **主装配唯一**：始终维护一个 `ROBOT_MAIN_ASM` 作为总入口。
2. **单向工作流**：Rhino → STEP → SolidWorks，禁止反向修改导致版本混乱。
3. **独立装配**：每个功能模块（如 AX1, AX2）应有独立的子装配体。
4. **版本控制**：所有改动必须进入 `REV` 或通过文件名版本号体现。
5. **发布规范**：发给工厂的文件统一打包在 `/07_MFG/RELEASE`。
