## 前置条件

你需要：

- Git（能 `git --version`）
- Python 3.8+（能 `python --version`）
- 一个 Gitee 或 GitHub 账号（用于提 PR）

<Tip>
  如果你是 Windows 用户：**推荐先用本地 Python + Git**，不要一上来就折腾虚拟机/WSL。真要走 Linux 路线，等你做完这个 PR 再升级也不迟。
</Tip>

## 时间线

<Steps>
  <Step title="0-10 分钟：克隆仓库 & 进入目录">
    ```bash
    git clone https://gitee.com/darrenpig/new_energy_coder_club.git
    cd new_energy_coder_club
    ```

    检查版本：

    ```bash
    git --version
    python --version
    ```
  </Step>

  <Step title="10-25 分钟：了解最小 Demo（无硬件）">
    选择一条当前可运行的入门路线：

    - **无硬件路线（推荐先完成）**：继续阅读本页，第 4 步会带你直接修改 `start-here/newcomers.mdx` 并提交 PR。这本身就是 NEC 工作流的最小可运行 Demo。
    - **有硬件路线**：如果你已有亚博 MicroROS-ESP32 套件或平衡小车，可参考下方「可选加速路线」直接跑通实体 Demo。

    无论哪条路线，完成后都请进入下一步：创建分支并做一个小贡献。
  </Step>

  <Step title="25-35 分钟：创建分支">
    建议分支命名（任选其一）：

    - `gfi/first-pr-<你的名字或ID>`
    - `docs/first-pr-<你的名字或ID>`

    ```bash
    git checkout -b gfi/first-pr-yourname
    ```
  </Step>

  <Step title="35-50 分钟：做一个必过审的小贡献">
    打开文件 `docs/start-here/newcomers.md`，在末尾追加一行：

    ```text
    @your_id｜方向：软件/硬件/机械/运营｜目标：完成第一个 good first issue｜设备：Windows / Mac / Linux｜时间：2025-12-24
    ```

    <Tip>
      也可以顺手做一个小改进：修正文档里的拼写，比如 Linux / Founder / EDA 等。
    </Tip>
  </Step>

  <Step title="50-60 分钟：提交 Commit + 发起 PR">
    **1) 检查改动**

    ```bash
    git status
    ```

    **2) 提交（建议用 Conventional Commits 风格）**

    ```bash
    git add docs/start-here/newcomers.md
    git commit -m "docs: add newcomer @your_id"
    ```

    **3) 推送到远端**

    ```bash
    git push -u origin gfi/first-pr-yourname
    ```

    **4) 在 Gitee/GitHub 上创建 PR**

    PR 标题建议：

    - `docs: add newcomer @your_id`
    - 或 `docs(start-here): add newcomer @your_id`

    PR 描述里写清楚：

    - 我已查看并尝试当前入门 Demo
    - 我新增了新人墙记录
    - 我想继续认领一个 `good first issue`
  </Step>
</Steps>

## 常见问题

<AccordionGroup>
  <Accordion title="python: command not found">
    - Windows：安装 Python 3（勾选"Add python to PATH"）
    - macOS：可以用 `python3` 代替 `python`

    ```bash
    python3 -c "print('hello NEC')"
    ```

    确认 Python 环境正常后，再按 [入门飞书文档](https://scn0bdoc8zxg.feishu.cn/wiki/QAtNwr244ir8ZekITEZcwpOZnkg) 运行当前 Demo。
  </Accordion>

  <Accordion title="找不到最小 Demo 入口">
    - **无硬件**：直接按本页第 4 步修改 `start-here/newcomers.mdx` 并提交 PR，这是零成本入门 Demo。
    - **有硬件**：参考下方「可选加速路线」中的 [MicroROS(ESP32)Robot](https://www.yahboom.com/study/MicroROS-ESP32) 或平衡小车资料。
    - 如果上述链接失效，请在社区群或 [Gitee Issues](https://gitee.com/darrenpig/new_energy_coder_club/issues) 中反馈。
  </Accordion>

  <Accordion title="我不知道下一步做什么">
    直接去 [First Good Issue 小白看这里](/start-here/first-good-issue)，从**文档类 good first issue**开始最稳。
  </Accordion>
</AccordionGroup>
